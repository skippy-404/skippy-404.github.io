Spring Boot 常用注解

一、 启动与核心配置注解

注解 说明 核心示例
@SpringBootApplication Spring Boot 启动类注解。一个复合注解，包含了 @SpringBootConfiguration、@EnableAutoConfiguration 和 @ComponentScan。是所有 Spring Boot 应用的起点。 java @SpringBootApplication public class MyApplication { public static void main(String[] args) { SpringApplication.run(MyApplication.class, args); } }
@EnableAutoConfiguration 启用自动配置。告诉 Spring Boot 根据你添加的依赖（jar包）来“猜测”并自动配置你想要的 Bean。例如，看到 spring-boot-starter-web 就会自动配置 Tomcat 和 Spring MVC。 (通常由 @SpringBootApplication 隐式包含)
@ComponentScan 组件扫描。默认扫描启动类所在的包及其所有子包下的组件（如 @Component, @Service 等）。 (通常由 @SpringBootApplication 隐式包含)
@Configuration 声明配置类。将一个类标记为 Spring 的配置源。这个类中可以使用 @Bean 注解来定义 Bean。 java @Configuration public class AppConfig { @Bean public RestTemplate restTemplate() { return new RestTemplate(); } }
@Bean 定义 Bean。用在方法上，告诉 Spring 这个方法的返回值是一个需要注册到容器中的 Bean。Spring 会负责管理这个对象的生命周期。 (见 @Configuration 示例)

二、 组件与依赖注入注解 (IoC/DI)

注解 说明 核心示例
@Component 通用组件注解。将一个类标记为 Spring 管理的组件。它是所有构造型注解（如 @Service, @Repository）的基础。 java @Component public class MyUtility { // ... }
@Service 业务逻辑层 (Service) 注解。语义上更清晰，表明这是一个业务服务类。功能上与 @Component 相同。 java @Service public class UserServiceImpl implements UserService { // ... }
@Repository 数据访问层 (DAO) 注解。用于标记数据访问组件。它除了具备 @Component 的功能外，还能将特定的数据访问异常（如 JDBC 异常）转译为 Spring 的统一数据访问异常。 java @Repository public interface UserRepository extends JpaRepository<User, Long> { // ... }
@Autowired 按类型自动注入。Spring IoC 容器会自动查找匹配类型的 Bean 并注入。这是最常用的依赖注入注解。 java @Service public class OrderService { @Autowired private UserService userService; // ... }
@Resource 按名称/类型注入。Java EE 的标准 (JSR-250)。默认按名称注入，如果找不到，则按类型注入。功能强大且灵活。 java @Service public class ProductService { @Resource private UserService userServiceImpl; // 注入名为"userServiceImpl"的Bean }
@Qualifier 指定注入的 Bean 名称。当同一类型有多个 Bean 实现时，与 @Autowired 配合使用，通过名称明确指定要注入哪一个。 java @Autowired @Qualifier("normalUserService") private UserService userService;
@Value 注入配置文件中的值。用于从 application.properties 或 application.yml 文件中读取配置值。 java @Value("${server.port}") private String serverPort;

三、 Web 开发注解 (Controller 层)

注解 说明 核心示例
@RestController 声明 RESTful 控制器。一个复合注解，包含了 @Controller 和 @ResponseBody。表示这个类中的所有方法返回的都是 JSON/XML 等数据，而不是视图（页面）。 java @RestController @RequestMapping("/api/users") public class UserController { // ... }
@Controller 声明传统 MVC 控制器。用于返回视图（如 Thymeleaf 或 JSP 页面）的场景。 java @Controller public class PageController { @GetMapping("/home") public String homePage() { return "index"; // 返回 index.html 视图 } }
@RequestMapping 路径映射。可以将 HTTP 请求映射到控制器的处理方法上。可以标注在类或方法上。 java @RestController @RequestMapping("/api/v1") // 类级别映射 public class ApiController { @RequestMapping("/status") // 方法级别映射 public String getStatus(){ return "OK"; } }
@GetMapping 处理 GET 请求。@RequestMapping(method = RequestMethod.GET) 的简写形式。 @GetMapping("/{id}")
@PostMapping 处理 POST 请求。@RequestMapping(method = RequestMethod.POST) 的简写形式。 @PostMapping("/create")
@PutMapping 处理 PUT 请求。@RequestMapping(method = RequestMethod.PUT) 的简写形式。 @PutMapping("/{id}")
@DeleteMapping 处理 DELETE 请求。@RequestMapping(method = RequestMethod.DELETE) 的简写形式。 @DeleteMapping("/{id}")
@PathVariable 获取路径参数。用于从 URL 路径中提取变量值（如 /users/{id} 中的 id）。 java @GetMapping("/{userId}") public User getUserById(@PathVariable Long userId) { // ... }
@RequestParam 获取查询参数。用于从 URL 的查询字符串中获取参数（如 /search?q=keyword 中的 q）。 java @GetMapping("/search") public List<User> search(@RequestParam("keyword") String keyword) { // ... }
@RequestBody 获取请求体数据。常用于 POST/PUT 请求，将请求体中的 JSON/XML 数据自动反序列化为 Java 对象。 java @PostMapping("/users") public User createUser(@RequestBody User user) { // ... }
@ResponseBody 将返回值序列化为响应体。将方法的返回值（通常是 Java 对象）自动序列化为 JSON/XML 格式，并写入 HTTP 响应体。@RestController 已默认包含此注解。 (通常在 @RestController 中隐式使用)

四、 数据访问与事务注解

注解 说明 核心示例
@Transactional 声明事务。可以标注在类或方法上。当标注在类上时，表示该类的所有 public 方法都将应用事务。可以配置隔离级别、传播行为等。 java @Service public class BankService { @Transactional(propagation = Propagation.REQUIRED) public void transfer(Long fromId, Long toId, BigDecimal amount) { // ...数据库操作 } }
@Entity JPA 实体类注解。表明这个类是一个实体类，它将映射到数据库中的一个表。 java @Entity @Table(name = "t_user") public class User { @Id private Long id; }
@Table 指定表名。与 @Entity 配合使用，指定实体类对应的数据库表名。 (见 @Entity 示例)
@Id 主键注解。指定实体类中的一个字段为数据库表的主键。 (见 @Entity 示例)
@GeneratedValue 主键生成策略。指定主键的生成方式，如自增 (IDENTITY)、序列 (SEQUENCE) 等。 @GeneratedValue(strategy = GenerationType.IDENTITY)
@Column 指定列名和属性。将实体类字段映射到数据库表的列，并可以定义列的属性（如名称、长度、是否允许为空等）。 @Column(name = "user_name", nullable = false, length = 50)

五、 辅助与增强注解

注解 说明 核心示例
@Slf4j (Lombok) 自动生成日志对象。由 Lombok 库提供，在编译时会自动为类生成一个名为 log 的 SLF4J 日志对象，省去手动编写 Logger log = LoggerFactory.getLogger(...) 的样板代码。 java @Slf4j @Service public class MyService { public void doSomething() { log.info("Doing something..."); } }
@Data (Lombok) ** Lombok 注解**。自动为类的所有字段生成 getter/setter 方法、toString(), equals(), hashCode() 和一个无参构造函数。 java @Data public class UserDTO { private String username; private String email; }
@ConfigurationProperties 类型安全的属性绑定。将配置文件 (application.yml) 中的一组属性值，自动绑定到一个 Java 类的字段上，实现强类型配置。 yaml my-app: name: "My Awesome App" version: "1.0.0" ``java @Component @ConfigurationProperties(prefix = "my-app") @Data public class AppProperties { private String name; private String version; }
