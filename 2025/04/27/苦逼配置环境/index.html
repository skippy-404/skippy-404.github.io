配模型环境的时候真的很想一拳打碎电脑屏幕

在实测各个i2i模型的风格化水平…
写这篇文章的时候，隔壁 safetensor 正在下载着…

有一说一，很多工具包，镜像源，已经极大简化了环境的配置流程，但是我从未顺利地一次性配置好模型环境过，由于都是在配置最新开源的模型，全网都找不到一个教程，其中踩过的坑，足矣写成经验贴，其中的委屈与心酸，足矣男默女泪。

4.25-4.26 bytedance_UNO

字节四月新发布的 i2i 模型——UNO ,宣传页中风格化的效果还是非常不错的，一定程度就能 work 我现在的任务，很巧的是前两天我刚买了一个skippy.uno的域名，抱着必须完成任务的信念，和对字节跳动这样顶尖组织的崇拜，我踏上了实验 UNO 的路。





对于这样刚发布的模型，对于是否有配置教程我是完全不抱有期待的，好在字节的 repo 中的文档整理的还算详细，模型和权重文件都是半自动化流程下载，很难想象及时是这样，我配置加测试也花了有整整两天。

  1. 没用过云端算力，花了些时间看 autodl 的使用教程，了解差不多了之后立即租了张 3090-24G 准备开干，信心满满地配置虚拟环境，直到下载模型的时候才发现默认下载到系统盘了，系统盘没办法扩容，这是第一败。

  2. ok，没关系，我新租了张 3090 开干，这次优化好了每一个路径，在漫长的等待之后，终于下载完了，命令行里兴奋且自信的输入 python inference.py ，显存又爆了，这时候已经有点烦躁了，王冶师兄给我说可能要租 48G 的卡，但是我租 3090 之前专门问了他，他说可以试一试，确实试了，确实败了，这是第二败。

  3. 总是重新开始配置肯定不行啊，于是我把系统盘镜像和数据盘全部传到阿里云盘，以后用别的卡直接挂载云盘就好了，kukuku… 传了半天，ok，也是用上好的了，这次直接租了一张 A40-48G，准备势在必得，但是 3090 的 cuda 版本 不适配 A40，我只好重新配置基础镜像，然后拿网盘的数据盘来用，kuku… 50多个G 传了半天，发现我如果加载不了之前的系统盘镜像，我的环境变量和路径全找不到了，之前的实例我全释放了，也看不到 shell 里的 history，这时候已经在考验我的耐心了，这是第三败。

  4. 重新开始，我释放了之前所有的卡，新租一张我们内蒙 A 区的 A40，在开机的时候，我给王冶哥默默发了一串消息，决心破釜沉舟

    漫长的下载时间过后，我终于跑通了，怀揣着喜悦的心情，我立马开始写脚本，准备批量生产测试效果，当脚本运行，看见数据面板 40G 显存被瞬间调用，那一刻，我仿佛能听见来自蒙古 A 区显卡的轰鸣，我心满意足地打开图片.. wdf，这是啥

第一时间，我真的觉得是我自己的问题，但是经过我反复测试之后，负责任地讲，截止当天，UNO 的风格化纯是骗我这种小丑，当天半夜 2：00 ，我气冲冲跑到字节的 github 上 issue 了这个问题，但是当我发现这个问题昨天就有人提，作者直接给人闭麦以后，懂得都懂了—— UNO这一场仗我大败特败 ，对我的折磨，说是直击内心深处的



4.27 Step-1X

早晨在社区在线测试了一下 Step-1X 的效果，还不错，然后尝试调用它的 api 批量测试，这个 api 文档真是我见到的里面足够草率的了，实例里的 url 都错了，响应体里还有 bug ？？最主要 api 只暴露了 medium 量级的模型，跟社区测试的效果天壤之别，没办法只能继续云端配置 Step-1X 模型的环境。

等一个下载可能就要半小时，一个报错就可能要重新进行很多下载，这样的帖子只会写这一次了，不光是回想这些记忆非常痛苦，而且写多了显得我蠢OvO。



写到这的时候又报错了,,,道阻且长


19:40 Step-1X 跑通了，效果不错很开心，可以去吃饭了！
害，效果还是很烂

4.29 FLUX_ControlNet

配置环境，还算顺利，

4.30-5.1 multi_control

验证 FLUX_ControlNet 的效果，开始面部与非面部解偶，multi inference 实现去风格，终于出了些效果，最近心情有点失落，明天重整旗鼓继续干吧

5.2 HiDream

下载 llama-3.1-8b-instruct 被 meta double reject，第一次随便填了一下申请，第二次填了 JLU，现在我挂 🇺🇸 的梯子，用当地的学校试一下，一个许可拖了我半天，lets fxxk meta!

nm 又被拒了，怀疑识别到我机器码了，服了 。。。
